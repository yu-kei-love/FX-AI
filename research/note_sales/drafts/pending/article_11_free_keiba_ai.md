# 初心者が競馬AIを作ったら回収率120%超えた話【Walk-Forward検証】

競馬AI。正直、自分みたいな素人が手を出す分野じゃないと思ってた。

Twitterで「競馬AI」と検索すれば山のように出てくるし、どれも難しそうな話ばかり。プログラミング経験ゼロの自分に何ができるんだろう、と。

でも、ひとつだけ気になることがあった。

世の中に溢れている競馬AIのほとんどが「的中率」を誇っている。的中率30%とか40%とか。でも、的中率だけでは儲かるかどうかわからない。10倍のオッズを当てれば10%の的中率でもプラスだし、1.2倍の本命を当て続けても控除率で負ける。

本当に大事なのは回収率、もっと厳密にいえばProfit Factor（PF）だと、調べていくうちにわかった。そしてそのPFを、Walk-Forward検証で正しく測定しているモデルがどれだけあるか。

ほとんどない。

素人だからこそ、逆に基本に忠実にやれるんじゃないか。そう思って始めてみた。

---

## 自己紹介

僕はプログラミング完全未経験から、AI（Claude）の力を借りて予測モデルを作り始めた人間だ。エンジニアでもデータサイエンティストでもない。コードはほぼAIに書いてもらっていて、理論を全部理解しているわけでもない。でも「検証で数字を出す」ことだけは徹底してきた。

今はFX、日本株、競馬、競艇、仮想通貨の5市場で予測モデルを動かしている。すべて同じフレームワーク――Walk-Forward検証＋アンサンブル学習――で作っている（この仕組み自体、Claudeに教えてもらった）。

この記事では、その中から競馬予測モデルの開発過程と検証結果を共有する。

---

## 典型的な競馬AIとの違い

まず、世の中の競馬AIの多くがどういうアプローチを取っているか。調べていくうちに見えてきた違いを整理してみる。

| 項目 | よくある競馬AI | このモデル |
|------|-------------|----------|
| 評価指標 | 的中率 | 回収率・PF |
| 検証方法 | ランダム分割 or 固定分割 | Walk-Forward（前進検証） |
| モデル | 単一モデル（LightGBMなど） | 5モデルアンサンブル |
| 特徴量 | 10〜20個程度 | 28個（pruning済み） |
| ベット戦略 | 予測上位に賭ける | EV閾値ベースのバリューベット |
| リスク管理 | なし or 固定額 | 日次・月次予算制限 |

一番の違いは検証方法だ。ランダム分割で検証すると、未来のデータが学習データに混入する（リーク）。結果、バックテストでは良い成績が出るが、実運用では使い物にならない。Walk-Forwardなら、常に過去だけで学習して未来を予測する構造になるので、リークを防止できる。

## モデルの概要

### アーキテクチャ：5モデルアンサンブル

単一モデルだと過学習のリスクが高い。性格の異なる5つのモデルを均等に組み合わせた。

| モデル | 特徴 |
|-------|------|
| LightGBM | 高速な勾配ブースティング。特徴量の重要度分析にも活用 |
| XGBoost | 正則化が強め。LightGBMとは異なるツリー構造を学習 |
| CatBoost | カテゴリ変数の処理に強い。過学習しにくい |
| RandomForest | バギング系。ブースティング系と異なるエラーパターンを捕捉 |
| ExtraTrees | ランダム分割でさらに多様性を確保 |

5モデルの予測確率を単純平均して最終予測としている。重み付けも試してみたけど、単純平均のほうが成績が安定した。Claudeに聞いたら「重みのチューニング自体が過学習の原因になりやすい」とのことで、なるほどと思った。

### 特徴量：28個（v2.1 pruned）

最初は30個以上の特徴量を入れていたけど、Claudeに「重要度分析と相関分析で不要なものを削ったほうがいい」と言われて、28個に絞った。

〈馬の基本情報：4個〉
- 馬齢、馬体重、体重変化、性別

〈成績関連：6個〉
- 勝率、複勝率、1走あたり賞金、前走からの日数
- 前走着順、直近3走平均着順

〈騎手関連：2個〉
- 騎手勝率、騎手複勝率

〈コース・条件：5個〉
- 枠番、距離、芝/ダート、馬場状態、出走頭数

〈レース属性：3個〉
- クラス（G1〜条件戦）、脚質（逃げ/先行/差し/追込）、人気順

〈オッズ関連：1個〉
- 単勝オッズ

〈交互作用・派生特徴量：7個〉
ここがモデルの肝になる部分だ。単独の数値だけでなく、複数の要素を掛け合わせた特徴量を作っている。

- ペース予測：逃げ馬・先行馬の割合からレースのペースを予測
- 脚質×ペース：ハイペースなら差し・追込有利、スローなら逃げ有利
- 距離適性：同距離帯での過去成績
- 馬場×体重：重馬場では体重が重い馬が有利な傾向
- 騎手×クラス：上位クラスでの騎手の実力差
- 枠番×距離：短距離芝では内枠有利
- クラス×馬齢：若い馬のクラス上昇パターン

v2.1で削除した特徴量も記録しておく。調教師勝率は騎手勝率と相関が高く、片方だけで十分だった。種牡馬エンコードと母父エンコードは重要度がほぼゼロ。血統情報はこのモデルでは効かなかった。Claudeに聞いたところ「サンプル数が不足しているのと、オッズに既に織り込まれているから」とのこと。

### 検証方法：Walk-Forward（前進検証）

Expanding Window方式のWalk-Forwardを採用した。

1. 最初の12ヶ月で学習 → 次の3ヶ月で予測
2. 最初の15ヶ月で学習 → 次の3ヶ月で予測
3. 最初の18ヶ月で学習 → 次の3ヶ月で予測
4. ...

学習データは常に過去方向に拡張していく。テスト期間は3ヶ月固定。これにより各Foldで「未来を知らない状態で予測する」ことが保証される。

## バックテスト結果

Walk-Forward検証の結果を公開する。

### 単勝

| 指標 | 値 |
|------|-----|
| 回収率 | 122.4% |
| PF（Profit Factor） | 1.56 |
| 的中率 | 8.7% |
| EV閾値 | 1.30 |
| ベット数 | 約650回 |

### 複勝

| 指標 | 値 |
|------|-----|
| 回収率 | 112.8% |
| PF（Profit Factor） | 1.29 |
| 的中率 | 26.3% |
| EV閾値 | 1.20 |
| ベット数 | 約920回 |

PF=1.56は「100円負けるごとに156円勝っている」という意味だ。控除率20〜25%の競馬で回収率120%超えは、かなり良い部類に入るらしい。

正直、この数字が出たとき自分でも驚いた。競艇モデル（PF=2.91）と比べるとかなり控えめだけど、Claudeに聞いたら「競馬は市場効率が高いので、この数字は十分良い」とのこと。Walk-Forwardで正しく検証しているからこそ言える数字だと思う。

### 重要な注意点

これはバックテスト結果だ。実運用では以下の理由でパフォーマンスが劣化する。

- オッズ変動：自分のベットでオッズが動く
- 控除率：JRAの控除率は約20〜25%。この壁を超え続けるのは容易ではない
- 市場環境の変化：新種牡馬の台頭、騎手の成績変動、コース改修など
- データ遅延：リアルタイムの馬体重や馬場状態の反映タイミング

PF=1.56は実運用ではPF=1.1〜1.3程度に落ち着くと想定している。それでもプラス圏なら十分だが、過度な期待はしないほうがいい。

## バリューベット戦略

このモデルの核心は「予測上位に賭ける」のではなく「期待値が閾値を超えた馬にだけ賭ける」ことだ。

### EVの計算

EV（Expected Value）= モデルの推定勝率 × オッズ

例えば、モデルが勝率15%と予測した馬のオッズが10倍なら：

EV = 0.15 × 10 = 1.50

EV > 1.0 なら理論上プラス期待値だが、控除率を考慮してEV > 1.30（単勝）を閾値に設定している。つまり「モデルが30%以上の優位性を感じた馬にだけ賭ける」戦略だ。

### なぜ人気馬に賭けないのか

1番人気の馬は勝率30%前後だが、オッズは2〜3倍程度。EV = 0.30 × 2.5 = 0.75 で、控除率を考えるとマイナス期待値になることが多い。

逆に、5〜10番人気でもモデルが実力を過小評価されていると判断すれば、高いEVが出る。こういう「市場が見落としている馬」を拾うのがバリューベットの本質だ。

## 特徴量エンジニアリングで気づいたこと

28個の特徴量のうち、いくつか面白い発見があった。素人なりに試行錯誤した結果なので、詳しい方から見たら当たり前かもしれないけど、共有しておく。

まず、ペース予測が想像以上に効いた。逃げ馬が多いレースではハイペースになり、差し・追込が有利になる。この傾向はデータにはっきり出ている。単純な脚質だけでなく「脚質×ペース」の交互作用を入れることで、Fold安定性が改善した。

次に、馬場×体重の交互作用。重馬場・不良馬場では体重の重い馬がやや有利になる。軽い馬場では体重はほとんど影響しない。この非線形な関係を交互作用項で表現している。

一方で、血統情報（種牡馬・母父）はほとんど効かなかった。最初は「血統って大事でしょ」と思って入れていたけど、効果がない。Claudeに相談したら「血統情報はオッズに既に織り込まれている。市場が知っている情報を入れてもエッジにならない」と教えてくれた。なるほど、と腑に落ちた。

## 正直にできていないこと

### うまくいった点

- 5モデルアンサンブルで単一モデルよりFold間の安定性が向上
- EV閾値ベースのバリューベットで、的中率は低いがPFは確保
- Walk-Forward検証でリークを防止し、現実に近い成績を測定
- 不要な特徴量のpruningで汎化性能を改善

### まだ解決できていない点

- 実運用での検証が不十分（バックテストのみ）
- 馬場状態の変化（レース中の雨など）に対応できていない
- 大レース（G1など）のサンプルが少なく、クラス効果の推定が不安定
- オッズ変動のシミュレーションを組み込めていない
- 控除率の壁（20〜25%）を安定的に超え続けられるか、長期では未検証

プログラミング未経験の自分でも、AIの力を借りればここまでは来れた。でも「Walk-Forwardで正しく検証する」「バリューベットで期待値を追う」という基本をきちんとやっているからこそ、数字が出ているのだと思う。的中率を競うのではなく、PFと回収率で勝負する。そこにこのモデルの意味がある。
