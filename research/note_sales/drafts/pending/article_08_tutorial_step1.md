〈ステップ3: はじめてのAI予測（15行で動く）〉

いよいよ本番です。メモ帳（Windowsの方）やテキストエディット（Macの方）を開いて、以下のコードをまるごとコピペしてください。

```python
import pandas as pd          # データを表で扱う道具
import yfinance as yf         # 株価を取ってくる道具
from lightgbm import LGBMClassifier  # 予測AIの本体

# トヨタの過去2年分の株価を取得する
df = yf.download("7203.T", period="2y")

# 前日比（何%動いたか）を計算する
df["Return"] = df["Close"].pct_change()

# 「翌日に上がったら1、下がったら0」という正解ラベルを作る
df["Target"] = (df["Return"].shift(-1) > 0).astype(int)

# AIが参考にする情報（特徴量）を作る
df["MA5"] = df["Close"].rolling(5).mean() / df["Close"] - 1   # 5日移動平均との差
df["MA20"] = df["Close"].rolling(20).mean() / df["Close"] - 1  # 20日移動平均との差
df["Vol"] = df["Return"].rolling(10).std()                      # 値動きの激しさ

# 空欄の行を消す
df = df.dropna()

# データを「学習用80%」と「テスト用20%」に分ける
X = df[["Return", "MA5", "MA20", "Vol"]]
y = df["Target"]
split = int(len(df) * 0.8)

# AIに学習させて、テストデータで正解率を出す
model = LGBMClassifier(n_estimators=100, verbose=-1)
model.fit(X[:split], y[:split])
accuracy = (model.predict(X[split:]) == y[split:]).mean()

print(f"正解率: {accuracy:.1%}")
```

このファイルを「predict.py」という名前でデスクトップに保存します。

保存したら、黒い画面で次のように打ってEnter。

Windowsの方:
```
python %USERPROFILE%\Desktop\predict.py
```

Macの方:
```
python ~/Desktop/predict.py
```

数秒待つと、結果が表示されます。

〈ステップ4: 結果を見る〉

画面に「正解率: XX%」と出ましたか？ おそらく52%〜58%くらいの数字が出るはずです。

「え、それだけ？」と思うかもしれません。でも、ちょっと考えてみてください。

コイン投げの的中率は50%です。もし55%と出たなら、AIはランダムより5%も良い判断をしているということ。株の世界では、ほんの数%の優位性があるだけで大きな差になります。

今回のコードで使った情報は、たった4つだけ。

| 情報           | 意味                             |
|---------------|----------------------------------|
| Return        | 当日の値動き                      |
| MA5           | 5日間の平均と今の価格のずれ         |
| MA20          | 20日間の平均と今の価格のずれ        |
| Vol           | 最近の値動きの激しさ               |

僕も最初にこの数字を見たとき「これって意味あるの？」と思いました。でも調べてみると、プロが使うAIモデルは50個、100個の情報を使うらしい。つまり、ここからまだまだ伸びしろがあるということ。この数字をどうやって改善していくかが、このシリーズのテーマです。
