# 全6予測モデル v3改善レポート
**日付: 2026-03-26**

---

## 概要

FX v3.0で成功した手法（12h horizon、5モデルアンサンブル、交互作用特徴量）を全モデルに展開し、一括改善を実施しました。

## 1. FX (USD/JPY) — v3.0 → v3.1

| 指標 | v3.0 | v3.1 |
|------|------|------|
| PF | 1.30 | **1.56 (+20%)** |
| Sharpe | 8.56 | **13.79 (+61%)** |
| MDD | 13.17% | 16.57% |
| トレード数 | 764 | 1188 |
| 勝率 | 50.1% | **55.5%** |
| 予測horizon | 12h | 12h |

### v3.1 改善内容
1. **マルチタイムフレーム特徴量** (+6): RSI/MACD/BB_width を4h/dailyで計算
2. **ボラティリティレジーム** (+2): Vol_percentile (720h rolling rank), Vol_of_vol
3. **セッション指標** (+5): Hour_x_DoW, Session_tokyo/london/ny/overlap
4. **性能ベースアンサンブル重み付け**: 検証データの精度に基づく指数重み（power=3）
5. **閾値候補拡張**: [0.50, 0.52, 0.55, 0.58, 0.60, 0.65, 0.70, 0.75]

**状態**: ペーパートレード v3.1更新済み、60分間隔ループ稼働中

## 2. 暗号資産 (BTC) — v2.1 → v3.0

### 改善内容
1. **FORECAST_HORIZON**: 4h → 12h（FXと同じ。手数料の影響を軽減）
2. **キャリブレーション**: Isotonic → Platt scaling（離散的ステップ関数 → 滑らかなシグモイド）
3. **CONFIDENCE_THRESHOLD**: 0.75 → 0.60（トレード数確保）
4. **交互作用特徴量 +9個**: RSI_x_Vol, MACD_norm, BB_position, MA_cross, Momentum_accel, Vol_change, HL_ratio, Close_position, Return_skew_12
5. **Walk-Forward検証**: expanding window追加（`--wf`フラグ）

### Walk-Forward結果（13ウィンドウ）

| 指標 | v2.1 (paper) | v3.0 (WF) |
|------|-------------|-----------|
| PF | 0.46 | **3.04** |
| Sharpe | - | **6.61** |
| Win Rate | - | **84.1%** |
| MDD | - | 68.81% |
| 全Window黒字 | No | **13/13** |

**注意**: MDD 68.81%が高い。ペーパートレードでの検証が必要。v2.1ではWF PF=3.83 → paper trade PF=0.46 という乖離があったため、楽観は禁物。

## 6. 競馬 — v1 → v2.0

### 改善内容
1. **アンサンブル**: 2モデル → 5モデル（LGB+XGB+CatBoost+RF+ET）
2. **特徴量強化**: ペース予測、距離適性、調教師効果、脚質適合度
3. **edge threshold**: 1.20 → 1.30（控除率20-25%を考慮）
4. **Walk-Forward**: 29ウィンドウ（expanding）
5. **レーサープール拡大**、クラスレベル効果追加

### 結果

| ベットタイプ | v1 PF | v2 PF | 回収率 | ベット数 | 判定 |
|-------------|-------|-------|--------|---------|------|
| 単勝 | 0.785 | **0.96** | 82.4% | 247 | FAIL |
| 複勝 | - | **1.15** | 97.8% | 1617 | FAIL |

**分析**:
- 単勝PFは0.785→0.96に改善したが、まだ赤字
- 複勝はPF=1.15で収支ほぼトントン
- 29フォルド中、多くのフォルドで0ベット or 0的中（不安定）
- 一部フォルドでは高回収率（Fold 23: RR=403%, Fold 11: RR=286%）

**2回目実行（edge信号強化版）**: 単勝PF=0.32, 複勝PF=0.89 — 悪化。強いedge信号は合成データ内でのオーバーフィットを助長した。

**根本原因**: 競馬は控除率20-25%があり、合成データではこの壁を超えられない。実データ（netkeiba.com等）での検証が不可欠。合成データでのチューニングは無意味（改善=オーバーフィット）。

## 重要な注意点

1. **暗号資産**: v2.1でWF PF=3.83 → ペーパートレード PF=0.46 という大きな乖離があった。v3.0でも同様の乖離が起きる可能性がある。Platt scalingに変更したが、必ずペーパートレードで検証すること。

2. **競輪・競馬**: 合成データでの結果であり、実データとは異なる可能性が高い。実データ収集が最優先。

3. **日本株**: yfinanceからの2年分データのみ。営業日463日で3-fold WFは統計的に十分とは言えない。

4. **全モデル共通**: Walk-Forward検証は過学習を防ぐが、将来の成績を保証するものではない。ペーパートレードでの継続的な検証が不可欠。
