# BoatraceAI - ボートレース予測システム 設計書

## 概要

LightGBMをベースとした3段階確率予測モデルです。
74種類の特徴量を用いて3連単確率を計算し、EV（期待値）が1.1以上の場合のみ購入シグナルを出します。

## 動作モード

`DataInterface` クラスで3つのモードを切り替えられます。

| モード | 用途 | データソース |
|--------|------|------------|
| `sqlite` | 本番・研究用 | SQLiteデータベース |
| `csv` | データ配布・共有用 | CSVファイル群 |
| `mock` | **デバッグのみ** | 合成データ（評価に使用禁止） |

```python
from data_interface import DataInterface

# SQLiteモード（本番）
di = DataInterface(mode="sqlite", db_path="data/boat/boatrace.db")

# CSVモード（データ共有）
di = DataInterface(mode="csv", csv_dir="data/boat/csv/")

# mockモード（デバッグのみ・評価に使用禁止）
di = DataInterface(mode="mock")
```

> **重要**: mockモードの合成データは動作確認のみ。予測精度・ROIの評価には実データのみ使用してください。

## 特徴量エンジン（74特徴量）

カテゴリA〜Jの74特徴量を計算します。

| カテゴリ | 内容 | 特徴量数 |
|----------|------|---------|
| A | 選手基本情報（勝率・級別・年齢等） | 8 |
| B | コース・スタート情報（**course_taken**・ST平均等） | 8 |
| C | コンディション（体重・チルト角等） | 4 |
| D | モーター・展示情報（モーター勝率・展示タイム等） | 6 |
| E | 環境（風速・風向・波高） | 5 |
| F | 会場（1コース勝率・海/淡水等） | 4 |
| G | レース構造（グレード・節内順番等） | 5 |
| H | オッズ時系列（7タイミング・変化率） | 14 |
| I | 対戦関係（H2H勝率・スタート差） | 8 |
| J | レース内相対値（ST偏差・オッズ偏差等） | 12 |

### 最重要特徴量：course_taken（実際の進入コース）

ボートレースでは**艇番（lane）と進入コースは別物**です。

- 前付け（コース入れ替え）が発生すると1号艇が6コースに入ることもある
- **`course_taken` = スタート展示後に確定した実際の進入位置**
- このシステムでは艇番ベースの特徴量は使わず、`course_taken`ベースで計算

## 3段階予測モデル

```
Stage1: 各艇の1着確率
  LightGBM + NN + ロジスティック回帰 → スタッキング
       ↓
Stage2: 条件付き2着確率
  「艇Aが1着の条件下で艇Bが2着になる確率」
       ↓
Stage3: 3連単120通りの確率計算
  全組み合わせのEVを計算
```

## EV計算と買い条件

```python
# 控除率25%を必ず考慮
EV = 予測確率 × オッズ × (1 - 0.25)

# EV ≥ 1.1 のみ購入候補
# 予測信頼度 ≥ 0.60 のみ購入候補
```

### オッズ急上昇フィルター（必須）

15分前オッズ → 1分前オッズの変化を監視：

| 変化率 | 処理 |
|--------|------|
| ≥ 20%上昇 | 買いシグナル完全取り消し |
| 10〜20%上昇 | 信頼度を50%カット |
| 5〜10%上昇 | 信頼度を20%カット |

> 「市場が知っていて自分が知らない情報がある」という前提で動作します。

---

## Kelly基準による賭け金計算

```python
# フルKelly × 0.25（保守的設定）
# 最大5%/レース上限
kelly_fraction = (prob * odds - 1) / (odds - 1) * 0.25
bet_size = min(kelly_fraction * capital, capital * 0.05)
```

### ドローダウン制御

| 状態 | 処理 |
|------|------|
| MDD ≥ 20% | 賭け金を半分に削減 |
| MDD ≥ 40% | 強制停止（RuntimeError） |

## データ取得

スクレイピング対象：`BOATRACE.jp`（公式サイト）

- 取得間隔：1〜2秒（サーバー負荷配慮）
- 取得済み日付は自動スキップ（重複防止）
- 保存先：SQLite（`data/boat/boatrace.db`）

オッズは7タイミングで取得：
`2時間前` → `1時間前` → `30分前` → `15分前` → `5分前` → `1分前` → `確定`
