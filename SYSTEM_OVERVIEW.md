## 1. システム概要

### 1.1 目的

FX（外国為替）と日本大型株の短期売買シグナルを、機械学習アンサンブルモデルで自動生成するシステム。
Telegram Bot経由でスマートフォンに通知し、承認/却下の手動確認を経てペーパートレードに記録する。

### 1.2 設計思想

- **リスク管理最優先**: 「くれぐれも大損のないようにリスク管理を最も大切にする」を全モジュールの基本方針とする
- **Walk-Forward検証必須**: バックテストだけでなく、未来データに対する逐次検証（Walk-Forward）を通過しなければ採用しない
- **5モデルアンサンブル**: 単一モデルへの依存を避け、LightGBM / XGBoost / CatBoost / RandomForest / ExtraTrees の多数決で安定性を確保
- **ファンダメンタル統合**: テクニカル指標だけでなく、経済サプライズ・CFTCポジション・原油価格・米国指数連動など、ファンダメンタル要素を特徴量に組み込む
- **保守的な閾値**: 高い自信度とモデル一致度を要求し、シグナル頻度よりも精度を重視する

### 1.3 想定資金と収益見込み（バックテストベース）

| 資産 | 年間見込み | ROI |
|------|-----------|-----|
| USD/JPY | +339,857円 | +34.0% |
| CAD/JPY | +502,567円 | +50.3% |
| ルネサス(6723.T) | +68,052円 | +6.8% |
| ソフトバンクG(9984.T) | +4,028円 | +0.4% |

※ 100万円資本、レバレッジ25倍（FX）の試算。実運用では結果が異なる可能性がある。

## 4. 株式予測モデル

### 4.1 スクリーニングプロセス

33銘柄（TOPIX100から流動性上位）に対して、13種類の米国指標（S&P500, NASDAQ, DOW, VIX, SOX, QQQ, XLF, XLE, XLV, 金, 原油, 銅, ドル指数）を組み合わせてテスト。合計429通りの組み合わせをWalk-Forward検証。

**採用基準**:
- プロフィットファクター (PF) >= 1.2
- コスト控除後期待値 > 0（コスト = 0.1%往復）
- 最低取引回数 >= 30
- Walk-Forward検証: 252日訓練 / 63日テスト（拡張ウィンドウ）

**戦略の考え方**: 米国市場は日本時間の深夜に閉まる。翌朝の日本市場は米国市場の動きに影響を受けやすい。この「オーバーナイト相関」を利用して翌日の方向を予測する。

## 5. リスク管理

リスク管理エンジン（`common/risk_manager.py`）が全トレードの前段に位置し、以下のルールを適用する。

### 5.1 ポジションサイジング

- **2%ルール**: 1トレードあたりの最大リスクは口座残高の2%
- 計算式: `lot_size = (残高 * 0.02) / |エントリー価格 - SL価格|`
- 連敗時は自動縮小（後述）

### 5.2 連敗保護

| 連敗数 | ポジションサイズ倍率 | 状態 |
|--------|---------------------|------|
| 0〜2 | 100% | 通常 |
| 3〜4 | 50% | 縮小 |
| 5〜6 | 25% | 大幅縮小 |
| 7以上 | 0%（取引停止） | 停止 |

### 5.3 損失制限

| 制限 | 閾値 | 動作 |
|------|------|------|
| 日次損失 | 残高の5% | 当日の新規取引を停止 |
| 週次損失 | 残高の10% | 当週の新規取引を停止 |
| ドローダウン | 最高残高の15% | サーキットブレーカー発動、全取引停止 |

### 5.4 相関ガード

- 相関係数0.75以上の通貨ペアは同方向での同時保有を禁止
- 例: EUR/JPYとGBP/JPYは相関0.90のため、両方ロングは不可
- 逆方向（ヘッジ目的）は許可

### 5.5 経済カレンダー

高インパクトイベント（2026年ハードコード済み）の前後で自動的に取引を停止する。

- **停止期間**: イベント4時間前〜2時間後
- **対象イベント**: FOMC（年8回）、NFP雇用統計（月1回）、CPI消費者物価（月1回）、BOJ会合（年8回）、ECB（年8回）、GDP速報（四半期）
- カスタムイベントのCSV読み込みにも対応

### 5.6 SL/TP設定（ATRベース）

| パラメータ | 値 |
|-----------|-----|
| ATR期間 | 14 |
| SL | 1.5 ATR |
| TP | 2.0 ATR |
| リスクリワード比 | 1 : 1.33 |
| 最大保有時間 | 24時間 |

- トレーリングストップ機能: 価格が有利な方向に動いた場合、SLを追従させる（ラチェット方式、引き下げなし）
- エグジット優先順位: SL > TP > 時間ベースエグジット

### 5.7 同時ポジション制限

- 最大同時ポジション数: 3（FX + 株を合算）

### 5.8 リスクジャーナル

全てのリスク判断（ブロック、取引開始、決済、日次/週次リセット、緊急停止/再開）をCSVに記録。
保存先: `data/paper_trade_logs/risk_journal.csv`

## 6. 自動研究パイプライン

`auto_research.py` が毎日深夜3時に自動実行され、新しい特徴量候補を検証する。

### 6.1 候補特徴量（8種類）

| 特徴量名 | ソース | 説明 |
|---------|--------|------|
| VIX_change | yfinance (^VIX) | VIX恐怖指数の変化率 |
| US10Y_change | FRED (DGS10) | 米国10年債利回りの変化 |
| Yield_spread_2y10y | FRED (DGS2, DGS10) | 2年-10年利回りスプレッド（景気後退指標） |
| Gold_change | yfinance (GC=F) | 金価格の変化率（安全資産フロー） |
| Oil_change | yfinance (CL=F) | 原油価格の変化率 |
| News_sentiment | NewsAPI | ニュースセンチメントスコア（-1〜+1） |
| DXY_change | yfinance (DX-Y.NYB) | ドルインデックスの変化率 |
| SP500_overnight | yfinance (^GSPC) | S&P500のオーバーナイトリターン |

### 6.2 検証プロセス

1. ベースデータ（USD/JPY 1時間足）を読み込み
2. 各候補特徴量を日足→1時間足にforward-fill展開
3. Walk-Forward検証（既存特徴量のみ vs 新特徴量追加）を比較
4. 採用基準を満たせばJSONファイルに記録（コード自動変更はしない）

### 6.3 採用基準

- PF > 1.3
- 期待値（手数料後）> 0
- 改善率 >= 5%（既存モデルからのPF改善）

### 6.4 安全ルール

- 信頼済みAPI（FRED, NewsAPI, yfinance）のみ使用
- 任意のURL・外部コード実行・ファイルダウンロード禁止
- 書き込みは `data/` ディレクトリのみ
- コードの自動変更は行わない（人間レビュー後に手動採用）

## 7. Telegram Bot

### 7.1 コマンド一覧

| コマンド | 説明 |
|---------|------|
| `/start` | 初回起動。チャットIDを保存 |
| `/status` | モデルの現在状態（予測回数、最新判断） |
| `/performance` | トレード成績（勝率、累積リターン、直近5件） |
| `/picks` | 本日の日本株ピックス |
| `/risk` | リスク管理状態（残高、DD、連敗数、日次/週次PnL） |
| `/help` | ヘルプメッセージ |

### 7.2 シグナル通知

シグナル発生時に以下の情報を通知する:
- カテゴリ（FX/株）、通貨ペア/銘柄名
- 売買方向（BUY/SELL）と現在価格
- 理由リスト（一致人数、自信度、レジーム、上昇確率、SL/TP）
- **インラインボタン**: 承認 / 却下 / 保留

承認ログは `data/paper_trade_logs/approvals.csv` に記録される。

### 7.3 AI会話機能

通常テキストメッセージにはClaude API（claude-sonnet-4-20250514）で応答する。
- 直近10往復の会話履歴を保持（Bot再起動でリセット）
- 200文字以内の簡潔な日本語回答
- 登録済みチャットIDからのメッセージのみ応答

## 8. 運用スケジュール

`run_all.py --loop` で以下のスケジュールが自動実行される。

| 時間帯 | タスク | 頻度 |
|--------|--------|------|
| 毎時 | FX予測（USD/JPY + CAD/JPY） | 1時間ごと |
| 朝 7〜9時台 | 株予測（ルネサス + ソフトバンクG） | 平日1回のみ |
| 深夜 3〜4時台 | 自動研究パイプライン | 1日1回 |
| 夕方 17〜18時台 | 日次パフォーマンスレポート（Telegram送信） | 1日1回 |

### 8.1 FX予測フロー（毎時）

1. `01_data_fetch.py` でデータ更新
2. 特徴量生成（テクニカル + 経済サプライズ + COTポジション）
3. 直近データで5モデルアンサンブル学習
4. 過去の予測結果を実際の値動きと照合
5. 最新の1本に対して予測
6. 経済カレンダー → リスク管理 → レジーム → ボラティリティ → 自信度 → 一致度の順でフィルタ
7. シグナル発生時はSL/TP計算 → ポジション登録 → Telegram通知
8. CSV記録

### 8.2 株予測フロー（日次）

1. yfinanceで日本株 + 米国指標の2年日足データ取得
2. 特徴量生成（米国リターン、VIX、為替、モメンタム、出来高、相関）
3. 5モデルアンサンブル学習（翌日上昇/下落の予測）
4. 過去予測照合
5. シグナル発生時はTelegram通知

## 9. 起動方法

### 9.1 メニューから起動（推奨）

```bat
start_system.bat
```

メニュー:
1. 全システム1回実行
2. FX自動ループ（1時間ごと）
3. 株のみ実行
4. Telegram Bot起動
5. 日次レポート送信
6. 全自動ループ（FX+株+レポート）

### 9.2 コマンドラインから起動

```bash
# 全システムを自動ループ（FX毎時 + 株日次 + 研究深夜 + レポート夕方）
python research/run_all.py --loop

# FXのみ自動ループ
python research/run_all.py --loop --fx-only

# 株のみ1回実行
python research/run_all.py --stock-only

# Telegram Bot起動（常駐）
python research/run_all.py --bot

# 日次レポート送信
python research/run_all.py --report
```

## 11. 重要な教訓

### 11.1 補間データの罠

5年分の1時間足データを補間（欠損値を線形補間）して使用した際、PF 3.84という異常に高い結果が出た。
しかしこれは補間データのパターンをモデルが記憶しただけであり、実データでは再現不可能だった。
**教訓**: 必ず実際のデータ（yfinanceの2年間の実時間足）のみを使用する。補間データは絶対に使わない。

### 11.2 テクニカル指標だけでは勝てない

初期バージョンではRSI/MACD/ボリンジャーバンドなどのテクニカル指標のみでモデルを構築したが、期待値がプラスにならなかった。経済サプライズ（FRED）やCFTCポジション（COT）などのファンダメンタル要素を加えて初めてPF > 1を達成した。
**教訓**: FX市場ではファンダメンタル要素が不可欠。テクニカルは補助。

### 11.3 相関が強い = 利益が出るではない

東京エレクトロン（8035.T）はSOX半導体指数との相関が+0.522と最も高かったが、Walk-Forward検証では基準未達だった。一方、相関がやや低いルネサス（+0.4263）は採用基準を満たした。
**教訓**: 相関の強さと予測可能性は別。Walk-Forward検証が唯一の判定基準。

### 11.4 現実: ほとんどの銘柄/通貨ペアでは勝てない

- FX: 8ペア中2ペアのみ利益が出る（USD/JPY, CAD/JPY）
- 株: 33銘柄中2銘柄のみ採用基準を満たす（ルネサス, ソフトバンクG）
- 採用率はわずか25%（FX）、6%（株）

**教訓**: 「勝てない」ことを認識し、勝てる少数のみに絞ることが重要。無理にトレードしないのが最善の戦略。

### 11.5 バックテストは将来を保証しない

全ての成績指標はバックテスト（Walk-Forward検証）の結果であり、実運用で同等の成績が出る保証はない。ペーパートレードで200回以上の取引を重ねてから、実資金投入の判断を行う。
