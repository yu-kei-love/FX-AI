# Product Spec

## 1. プロダクト名
FX AI 自動売買システム（初期対象: USD/JPY）

## 2. 目的
USD/JPY を対象に、時系列リーク対策と過学習管理を備えた FX AI 学習・検証・可視化システムを構築する。
初期段階では実弾売買を行わず、バックテスト・検証・ペーパートレードまでを完成させる。

## 3. 解決したい課題
- 個人開発でも現実的に作れる FX AI 開発基盤を作る。
- 見かけだけのバックテストではなく、リーク対策・過学習対策を先に実装する。
- モデルを増やす前に、執行コスト・見送り判断・停止条件を組み込む。
- 学習過程・検証結果・相場状態を可視化できる状態にする。

## 4. 初期対象スコープ
### 対象
- 通貨ペア: USD/JPY
- 実行対象: データ収集、特徴量生成、ラベリング、学習、検証、可視化、ペーパートレード
- UI: Streamlit ダッシュボード
- 保存: ローカルファイル中心。必要最小限でローカルDBを補助的に使うのは可

### 非対象
- 実弾自動売買
- 複数通貨ペア同時運用
- 高頻度取引
- MARL / HRL のような高度強化学習
- フル感情分析中心のシグナル生成

## 5. 入力データ
### Phase 1 で必須
- OANDA API: FX価格データ、スプレッド

### Phase 4 以降で拡張候補
- FRED API: マクロ経済系列
- Alpha Vantage: 経済指標カレンダー関連情報
- NewsAPI: ニュース時刻ベース特徴量

## 6. 主要機能
1. 価格データ取得
2. スプレッド取得・保存
3. データクレンジング
4. 特徴量生成
5. HMM によるレジーム判定
6. LightGBM による方向予測
7. Triple-Barrier Labeling
8. Meta-Labeling
9. Purged K-Fold CV
10. Walk-Forward 評価
11. PBO / DSR 評価
12. No-Trade 判定
13. 執行コスト考慮バックテスト
14. 異常停止・監視
15. 可視化ダッシュボード
16. ペーパートレード準備

## 7. 出力
- 学習済みモデル
- 特徴量重要度
- バックテスト結果
- Walk-Forward 結果
- PBO / DSR 指標
- 売買シグナル
- No-Trade 判定
- 停止ログ
- 可視化グラフ

## 8. 成功条件
### Phase 1 成功条件
- OANDA から USD/JPY データを取得できる
- 基本特徴量を生成できる
- HMM と LightGBM が動く
- 結果を最低限可視化できる

### Phase 2 成功条件
- Triple-Barrier / Purged CV / Walk-Forward / PBO / DSR が動く
- 過学習の少ない検証基盤を作れる

### Phase 3 成功条件
- Meta-Labeling と No-Trade 判定が動く
- 異常停止と運用監視が動く

### Phase 4 成功条件
- ニュース時刻特徴量と Optuna による最適化が動く
- レジーム別モデル切替の基礎が動く

## 9. 設計原則
- 初心者でも理解できる構成
- モジュール責務を明確に分離
- 将来拡張しやすいが、初期実装は小さく保つ
- 検証を先、複雑モデルは後
