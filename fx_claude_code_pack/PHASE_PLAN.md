# Phase Plan

## Phase 1: 最小学習基盤
### 目的
最小のデータ取得・特徴量生成・レジーム判定・基本予測パイプラインを構築する。

### 実装対象
- OANDA API 接続
- USD/JPY OHLC + スプレッド取得
- 生データ保存
- 基本特徴量生成
- HMM レジーム判定
- LightGBM 基本モデル
- 最低限の可視化

### 完了条件
- データ取得が成功する
- データ保存が成功する
- HMM が動く
- LightGBM が学習・推論できる
- 結果グラフが出る

### 非対象
- Meta-Labeling
- PBO / DSR
- ニュース特徴量
- PPO

## Phase 2: 検証基盤強化
### 目的
見かけの成績ではなく、リーク対策と過学習管理を備えた検証基盤を作る。

### 実装対象
- Triple-Barrier Labeling
- Purged K-Fold CV
- Walk-Forward 評価
- PBO
- DSR
- 執行コストモデリング

### 完了条件
- リーク対策済み評価が動く
- PBO / DSR を計算できる
- コスト込み評価が出る

## Phase 3: 見送り判断と監視
### 目的
「エントリーする」だけでなく「見送る」「止める」ができる状態にする。

### 実装対象
- Meta-Labeling
- No-Trade 判定
- 重要指標前後停止
- スプレッド閾値停止
- 日次損失停止
- 連敗停止
- 異常検知ログ

### 完了条件
- 採用可否モデルが動く
- 見送り判断が機能する
- 停止条件が動く

## Phase 4: 特徴量拡張と最適化
### 目的
追加特徴量と自動最適化で改善余地を検証する。

### 実装対象
- ニュース時刻ベース特徴量
- Alpha Vantage / FRED の特徴量統合
- Optuna による最適化
- レジーム別モデル切替の基礎実装

### 完了条件
- 追加特徴量が学習に組み込まれる
- Optuna が動く
- レジーム別切替が試せる

## Phase 5: ポジションサイズ最適化
### 目的
方向予測と分離した形でポジションサイズ調整を強化学習で試す。

### 実装対象
- PPO の限定導入
- ペーパートレード
- サイズ調整ロジック

### 完了条件
- 仮想資金で安定して動く
- 方向予測部分を壊さずサイズ調整を実験できる

## Phase 6: 研究拡張
### 目的
有意差が確認できた場合のみ高度モデルを検証する。

### 実装対象候補
- TFT
- HRL
- MARL

### 完了条件
- 既存構成より統計的に意味のある改善がある場合のみ採用

## フェーズ共通ルール
- 各 Phase は必ず前 Phase の完了条件を満たしてから進む。
- 成績がよく見えてもリーク疑いがあれば不採用。
- 高度モデルを先に作らない。
